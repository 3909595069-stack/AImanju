# 02 · 角色设计 — 给 AI 一份不会弄丢的"视觉身份证"

> 你说"保持一致"，AI 说"你说的是谁？"——角色设计就是给 AI 一份视觉身份证，让同一个角色在所有镜头里长得一模一样。

## 🔍 这是什么？

**角色设计**，在 AI 短剧工作流里特指：把文字剧本里的角色，转化为一套 **AI 能稳定复现的视觉标准**。

这个概念源自动画工业的「角色三视图」（Character Design Sheet / Model Sheet）：

| 视图 | 作用 | 在 AI 工作流中对应什么 |
|:--|:--|:--|
| **正面图** | 脸部特征、发型、五官比例 | 即梦/ Midjourney 生成的正面定妆照 |
| **侧面/半侧图** | 头型、鼻梁、下颌线轮廓 | 同一角色的多角度生成图 |
| **背面图 + 全身** | 体型、服装、站姿、身高比例 | 用于全景镜头的人物参考 |
| **表情集** | 喜怒哀乐的不同状态 | 用于特写镜头、情感戏的参考 |

> **一句话总结**：角色设计 = 选定角色外貌 → 固化为 3-5 张参考图 → 在所有分镜中锚定这个形象。

## 💡 为什么这是最关键的一步？

整个 AI 漫剧工作流里，**这一步决定了你后面所有画面的质量上限**。原因就一个：

**AI 没有记忆。** 你上一张图生成一个黑发红裙的女孩，下一张它完全不记得她长什么样。你在人眼里是「同一个人」，在 AI 眼里每次都是重新掷骰子。

### 不做角色设计的典型翻车现场

- 第一幕是瓜子脸，第二幕变成圆脸 → 弹幕「这谁？」
- 衣服颜色每个镜头都在变 → 弹幕「换装秀？」
- 发型飘忽不定，刘海一会儿有一会儿没有 → 弹幕「发型比剧情变化还快」
- 眼睛颜色从紫色变成蓝色再变成棕色 → 观众直接弃剧

> 角色设计的目的不是"画出好看的人"，而是**锁死变量**——让 AI 每次画这个角色时，输入条件完全一致。

## 🛠️ 怎么做？（详细操作）

### 方法一：即梦参考图法（零门槛，推荐新手）

#### 第 1 步：生成角色定妆照

打开 **即梦 AI**（jimeng.jianying.com，抖音号登录）→「图片生成」→ 模型选「图片 4.0」→ 比例选「9:16」→ 输入提示词：

```
[性别][年龄段]，[脸型]，[发型+发色]，[眼睛特征]，[肤色]，
穿着[服装描述]，[配饰]，[气质/表情倾向]，
[画风关键词]，[光线]，纯白背景，角色设定图，三视图
```

**完整示例（古风女主）：**
```
年轻女性，22岁左右，鹅蛋脸，黑色长直发及腰，齐刘海，
紫色杏眼双眼皮，雪白肤色，
穿着白色古风长裙，红色腰带，袖口金色刺绣，腰间挂白玉佩，
清冷疏离气质，眉眼间带着淡淡忧愁，
二次元赛璐璐风格，柔光，纯白背景，角色设定图，三视图，全身站立
```

**完整示例（都市男主）：**
```
年轻男性，25岁左右，轮廓分明的方脸，深棕色短发微乱，
深褐色眼睛眼神锐利，小麦色肤色，
穿着洗旧的白色T恤、深蓝色牛仔裤、黑色马丁靴，
右耳戴一颗银色耳钉，下巴有淡淡胡茬，
痞帅但眼神温柔，有市井烟火气，
韩式条漫风格，自然光，纯白背景，角色设定图，全身站立
```

**完整示例（英文二次元）：**
```
3D chibi character, silver short hair, sharp blue eyes,
oversized grey hoodie with cat ear design, black fingerless gloves,
young male, slim build,
three views (front, side, back), full body standing pose,
clean white background, anime style, cel shading, high quality, 8K --ar 16:9
```

> 💡 多生成几次（每次结果不同），挑最满意的一张保存为 `角色名_定妆照.png`

#### 第 2 步：建立多角度参考图库

有了满意的正面照后，生成该角色的其他角度：

1. 在即梦图片生成界面，点击输入框左边的 **上传参考图按钮**
2. 上传刚才的正面定妆照
3. 把提示词中的「正面站立」改为「侧面站立（身体朝右）」→ 生成侧面图
4. 再改为「背对镜头，背面站立」→ 生成背面图
5. 生成一张「面部特写，表情自然，从胸口到头顶」→ 近景表情参考

#### 第 3 步：建立「角色身份证」文档

把所有参考图存入同一个文件夹，命名规则 `[角色名]_定妆`，包含：
- `[角色名]_正面.png`
- `[角色名]_侧面.png`
- `[角色名]_背面.png`
- `[角色名]_表情特写.png`
- `[角色名]_全身.png`

#### 第 4 步：在所有分镜中使用

后续每个分镜生图时，**都上传这些参考图**，并在提示词中重复角色的核心外貌关键词。

### 方法二：Midjourney --cref 方案（画质最好，需付费）

Midjourney 是目前角色一致性最成熟的付费方案。

#### 关键词参数速查

| 参数 | 作用 | 设置范围 | 推荐值 |
|------|------|---------|--------|
| `--cref [URL]` | Character Reference：锁定人物外观 | 上传 1-3 张图片 URL | 2 张（正面 + 侧面） |
| `--cw` | Character Weight：控制参考影响程度 | 0-100 | 日常 50-80，换装时 0-30 |
| `--sref [URL]` | Style Reference：锁定画风和色调 | 上传 1 张风格图 URL | 1 张风格图 |
| `--seed [数字]` | 固定随机种子 | 任意整数 | 记录首张满意图的 seed 值 |

#### 操作步骤

1. **生成初始角色**：
```
a young female warrior, silver hair, red eyes, black armor,
front view, full body, character design sheet, white background --v 7.0 --ar 16:9
```

2. **用 --cref 锁角色，生成不同角度**：
```
side view of the same character --cref [正面图的URL] --cw 80 --v 7.0
back view of the same character --cref [正面图的URL] --cw 80 --v 7.0
```

3. **用 --sref 锁风格**：
```
the warrior standing in a bamboo forest, morning mist --cref [角色URL] --sref [风格图URL] --v 7.0
```

4. **记录 seed 值**：首张满意角色图生成后，用 Discord 的 ✉️ 反应获取 seed 值，后续所有相同角色的生成都加上 `--seed [该值]`

> 💡 **Midjourney 小贴士**：
> - `--cw 0` = 只锁脸，可以换衣服（「换装不换脸」）
> - `--cw 100` = 锁脸 + 锁衣服 + 锁发型（全套锁定）
> - V7 的 `--oref`（Omni Reference）比 V6 的 `--cref` 更好用，但消耗 2 倍 GPU 时间

### 方法三：ComfyUI + IP-Adapter 方案（专业级，免费但需显卡）

ComfyUI 是目前最强大的免费 AI 绘画工具，IP-Adapter 是其中锁定角色一致性的核心插件。

#### 需要什么硬件

- **最低**：8GB 显存（NVIDIA RTX 3060 以上）
- **推荐**：12GB+ 显存
- **安装**：github.com/comfyanonymous/ComfyUI → 下载一键安装包

#### IP-Adapter 核心参数设置

| 参数 | 作用 | 推荐值 | 说明 |
|------|------|--------|------|
| **weight（权重）** | 控制角色参考图的影响力 | 0.6-0.8 | 面部特写用 0.8，全身用 0.6，动漫用 0.4-0.7 |
| **noise（噪声）** | 允许表情微变化的程度 | 0.2 | 太高变脸，太低表情僵硬 |
| **clip_vision 模型** | 图像编码器 | ViT-H-14 | 兼容性最好 |
| **start/end** | 影响时段 | 0.0-1.0 | 全程影响 |

#### 三件套叠加方案（工业级标准）

```
第一层：LoRA（锁定角色身份 + 画风）— 强度 0.6-0.8
    ↓
第二层：IP-Adapter FaceID（锁定面部特征）— 权重 0.6-0.8
    ↓
第三层：ControlNet OpenPose（锁定姿势骨架）— 权重 0.7-0.9
    ↓
输出：角色长得一样 + 姿势可以自由变化
```

> 这个三层叠加方案，角色一致性可达 **90-95%**，是目前社区公认的工业级标准。

---

### 方法四：训练专属 LoRA（终极方案，角色 IP 化运营）

当你的角色要在 **50 集以上** 的连载剧中反复出现，或者你要做角色 IP 化运营时，训练一个专属 LoRA 是最佳选择。

#### 数据集准备

| 要求 | 标准 |
|------|------|
| **图片数量** | 15-30 张高质量图（非越多越好，质量 > 数量） |
| **角度覆盖** | 正面 ×2-3，3/4 侧面 ×4-5，侧面 ×2，仰视/俯视 ×2-3 |
| **表情多样** | 自然 ×3-4，微笑 ×3-4，严肃 ×2-3 |
| **距离变化** | 脸部特写 ×5-7，头像 ×5-7，半身 ×3-4，全身 ×2-3 |

#### ⚠️ 数据集中**不要**包含的图片

- 戴墨镜 / 面部被手或物体遮挡的
- 加了重滤镜的（美颜、柔焦）
- 运动模糊或对焦不准的
- 极端妆容和极端表情的（会学歪）

#### 关键训练参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| **Network Rank** | 32-64 | 32 起步，更高捕获更多细节但易过拟合 |
| **Network Alpha** | 16-32 | 通常设为 rank 的一半 |
| **学习率** | 1e-4（0.0001） | 稳定起点 |
| **训练轮数** | 10-15 epochs | 15-30 张图的最佳点 |
| **Batch Size** | 1-2 | 取决于显存 |
| **优化器** | AdamW8bit 或 Prodigy | 节省显存 |
| **训练分辨率** | 1024×1024（SDXL）或 512×512（SD1.5） | |
| **推荐基础模型** | Pony Diffusion V6（动漫） / Juggernaut XL（写实） | |

#### 怎么判断训练成功了？

- ✅ 生成图与训练图**像同一个人但不是同一张照片**（既有辨识度又有变化）
- ✅ 触发词能稳定召唤这个角色
- ✅ 可以生成训练集里没有的姿势和表情
- ❌ 生成图与训练图一模一样 → **过拟合了**，降低 rank 或增加数据集多样性
- ❌ 触发词无效，角色看起来通用 → **欠拟合了**，增加 epochs 或学习率

## 📊 四种方案完整对比

| 方案 | 原理 | 一致性 | 门槛 | 成本 | 适用场景 |
|:--|:--|:--|:--|:--|:--|
| **即梦参考图法** | 即梦内置参考功能 | ~70% | ⭐ 零门槛 | 免费 | 1-3 分钟短视频/个人练习 |
| **Midjourney --cref** | `--cref` 参数锁角色 | ~85% | ⭐⭐ | $30/月起 | 中高质量短剧/已有 MJ 订阅 |
| **ComfyUI IP-Adapter** | IP-Adapter 注脸 + ControlNet 控姿 | ~90% | ⭐⭐⭐ | 免费(需显卡) | 高质量系列剧/有技术能力 |
| **训练专属 LoRA** | 定制轻量模型 | ~95%+ | ⭐⭐⭐⭐ | 5-20 元/LoRA | 角色 IP 化运营/50 集以上连载 |

> **新手建议**：用即梦参考图法入门，做到第 3-5 个作品时，你会自然感受到瓶颈，再升级到 Midjourney 或 ComfyUI。

## 🎯 好角色设计的 3 个标准

### 1. 特征鲜明且有记忆点

| ✅ 好的设计 | ❌ 差的设计 |
|:--|:--|
| 「红框圆眼镜 + 右眼角泪痣 + 不对称短发」 | 「一个好看的女生」 |
| 「左脸有一道从眉毛到下巴的暗红伤疤」 | 「忧郁气质的男主角」 |
| 「永远戴一条银色十字架项链」 | 「时尚穿搭」 |

> **原则**：AI 更容易锚定**具体的、可文本描述的特征**，而非模糊的美感。

### 2. 跨角度可辨识

**自我测试**：把正面、侧面、背面三张图混在 10 张其他角色图中，让朋友找出同一个人。

跨角度辨识的三大核心要素：
- **发型轮廓**（从背面也能认出来）
- **服装颜色搭配**（从远处也能分辨）
- **标志性配饰**（从任何角度看都一样的物件）

### 3. 与故事基调匹配

- 古风角色 → 别用赛博朋克配色
- 悬疑剧角色 → 避免过于甜美的粉色系
- 热血少年漫角色 → 避免过于阴柔的服装和妆容

> 角色设计是**第一层视觉叙事**——观众在看到故事之前，已经通过角色形象判断了这是什么样的故事。

## 🐛 做坏了怎么办？

| 问题 | 症状 | 诊断 | 解决方案 |
|:--|:--|:--|:--|
| **脸盲症** | 每张图长得不一样 | 参考图未正确上传或权重不够 | ① 确认参考图已上传 ② 减少发型/服装的变化描述 ③ 在提示词开头用大写重复角色名 |
| **服装漂移** | 衣服颜色/款式变了 | 服装关键词不够强 | 在正向提示词中**三次重复**关键服装要素（如"红色腰带，红色腰带，红色腰带"） |
| **画风不统一** | 一张写实一张二次元 | 没有锁定画风关键词 | 把画风关键词放在提示词**最前面**，确保所有角色的参考图风格一致 |
| **表情僵硬** | 所有图都是同一个表情 | 角色定妆照是中性表情，但没加表情词 | 在生图时加入具体情绪描述：「嘴角微扬，眼神柔和」「眉头紧锁，嘴唇紧抿」 |
| **侧脸崩** | 正面像但侧脸不像同一个人 | 参考图只有正面 | 补充生成侧面参考图，或把正面+侧面两张图同时上传为参考 |
| **多角色混淆** | A 的脸出现在 B 身上 | 两个角色同时出现时 AI 混淆了 | 每个角色分开发送参考图，在提示词中明确标注「图左：[角色A描述]；图右：[角色B描述]」 |
