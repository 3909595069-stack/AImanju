# ⚙️ ComfyUI + IP-Adapter 进阶教程

## IP-Adapter 是什么？

IP-Adapter（Image Prompt Adapter）是一个插件，它能把一张参考图的视觉特征"注入"到生成过程中。不像 LoRA 需要训练，IP-Adapter 是**即插即用**的——给它一张图，它就把这张图的面部特征挂到你的生成上。

## 核心参数速查

| 参数 | 推荐值 | 作用 |
|------|--------|------|
| weight | 0.6 - 0.8 | 参考图影响力。面部特写 0.8，全身 0.6 |
| noise | 0.2 | 允许表情微变化，不改变核心特征 |
| clip_vision | ViT-H-14 | 图像编码模型 |
| CFG | 7.0 | 提示词遵循度 |
| steps | 30 | 采样步数 |
| sampler | dpmpp_2m | 采样器 |

## 工业级三件套叠加方案

这是目前社区公认的**最强角色一致性**方案：

```
第一层：角色 LoRA（锁身份 + 画风）
    ↓ 强度 0.6 - 0.8
第二层：IP-Adapter FaceID（锁面部特征）
    ↓ 权重 0.6 - 0.8
第三层：ControlNet OpenPose（锁姿势骨架）
    ↓ 权重 0.7 - 0.9
输出：角色一致 + 姿势自由 + 场景自由
```

### 工作原理

- **LoRA** 负责「这是谁」（五官、体型、画风）
- **IP-Adapter** 负责「脸是不是同一个人」（从参考图提取面部特征向量）
- **ControlNet** 负责「姿势和构图」（骨骼图、线稿）

三者各管一摊，互不冲突。这就是为什么这个方案比单独用任何一种都要强。

## IP-Adapter 权重调节指南

| 情况 | weight 建议 | 说明 |
|------|-----------|------|
| 面部特写镜头 | 0.8 - 0.85 | 脸占画面大部分，需要强锁 |
| 半身/中景 | 0.7 | 标准值 |
| 全身/远景 | 0.5 - 0.6 | 脸小，太强会让角色僵硬 |
| 动漫风格 | 0.4 - 0.7 | 动漫对面部精度要求低于写实 |
| 换姿势但保脸 | 0.5 - 0.6 | 降低权重给姿势留空间 |
| 多人场景 | 每个角色单独的 IP-Adapter 分支 | 避免角色间特征混淆 |

## LoRA 训练参数速查

如果决定训练专属 LoRA：

| 参数 | 推荐 |
|------|------|
| 图片数量 | 15-30 张 |
| Network Rank | 32-64 |
| Network Alpha | 16-32（rank 的一半） |
| 学习率 | 1e-4 |
| Epochs | 10-15 |
| Batch Size | 1-2 |
| 优化器 | AdamW8bit 或 Prodigy |
| 训练分辨率 | 1024×1024（SDXL）|

### 训练数据怎么准备？

| ✅ 应该包含 | ❌ 不要包含 |
|:--|:--|
| 正面 ×2-3 | 墨镜/面部遮挡 |
| 3/4 侧面 ×4-5 | 重滤镜/美颜 |
| 侧面 ×2 | 运动模糊 |
| 仰视/俯视 ×2-3 | 极端妆容 |
| 自然表情 ×3-4 | 角度单一（全是正面） |
| 微笑 ×3-4 | 分辨率太低（<512px） |
| 严肃 ×2-3 | |

### 训练失败怎么判断？

| 症状 | 诊断 | 解决 |
|------|------|------|
| 生成的图和训练图一模一样 | 过拟合 | 降低 rank 到 16，增加数据集多样性 |
| 触发词无效，角色长得随机 | 欠拟合 | 增加 epochs 到 15-20，降低学习率 |
| 只能生成训练集中的姿势 | 数据多样性不够 | 补充不同角度和姿势的训练图 |
| Loss 降到 0.03 以下又反弹 | 过拟合的典型信号 | 停止训练，用更早的 checkpoint |
