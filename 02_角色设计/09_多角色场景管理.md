# 👥 多角色场景管理

> 当画面里出现两个以上的角色时，AI 最容易翻车——角色混淆、特征串位、某人消失不见。这是 AI 漫剧进阶路上必须攻克的山头。

## 核心问题：为什么多人场景这么难？

AI 图像生成的核心机制是"从噪声中一步步还原出画面"——这和画家从空白画布开始画不同。AI 没有一个"角色 A 在左边、角色 B 在右边"的全局概念，它是基于文字提示去"猜测"画面里有什么。

**结果**：当提示词里出现两个人物描述时，AI 经常把两人的特征混在一起（角色 A 的发色出现在角色 B 头上），或者干脆少画一个人。

## 实战解决方案

### 方案一：分区描述法（最可靠）

在提示词中明确标注空间位置：

```
图左（画面左侧）：[角色A完整描述]
图右（画面右侧）：[角色B完整描述]

两人面对面站立，之间距离约1米。
角色A面向右侧，角色B面向左侧。

背景：[场景描述]
```

**完整示例**：
```
图左（画面左侧）：年轻女性，黑色长直发，紫色瞳孔，白色古风长裙，红色腰带，面向右侧，侧脸对镜头。
图右（画面右侧）：年轻男性，银白长发束起，深蓝眼眸，玄色劲装，面向左侧，侧脸对镜头。
两人面对面站立，相距约1米，气氛紧张对峙。
背景：竹林深处，黄昏逆光，金色轮廓光。
```

### 方案二：分角色生成 + 合成（最稳妥）

1. 分别生成只有角色 A 和只有角色 B 的分镜图
2. 用图片编辑工具（PS / Canva / Photopea）把两个角色合成到同一张背景图上
3. 把合成后的图作为"参考图"上传给即梦，再进行一次整体的图生图

这个方案**绝对保证角色一致性**，就是多一个步骤。

### 方案三：即梦"批量生成 + 人工筛选"

即梦在多人场景下的成功率大约 30-50%。所以要靠数量取胜：
- 每次生成 4-8 张
- 从中筛选没有角色混淆的
- 一批不行就微调提示词再生成一批

### 方案四：商汤 Seko 2.0（2025年底新方案）

商汤科技发布的 Seko 2.0 是业界首个支持**多角色精准管理**的智能体：
- **SekoIDX 技术**：跨分镜、多剧集保持角色外貌一致
- **SekoTalk 技术**：业内首个支持**超过 2 人精准对口型**的方案
- 适合做长篇多角色群像剧的创作者关注

## 多人场景的景别选择

| 角色人数 | 推荐景别 | 原因 |
|---------|---------|------|
| 2 人 | 中景/近景 | 能看清两人的表情互动 |
| 3 人 | 中景/全景 | 需要更宽的取景范围 |
| 4 人+ | 全景/远景 | 单个角色的面部细节不再重要，重在站位关系 |

## 多对话镜头的"正反打"技巧

影视常用的"正反打"可以避免多人画面的技术难题：

```
镜头1（近景·固定）：角色A说话 → 画面里只有A，B在画外
镜头2（近景·固定）：角色B回应 → 画面里只有B，A在画外
镜头3（中景·侧面）：两人同框 → 用这个镜头交代两人的空间关系
镜头4（特写·A）：A的情绪反应
镜头5（特写·B）：B的情绪反应
```

> 💡 **核心技巧**：90% 的对话用单人镜头（正反打）+ 10% 的双人全景建立空间感。这样既避免了多人场景的 AI 翻车风险，也让剪辑节奏更有电影感。

## 多角色管理清单

| 管理维度 | 具体做法 |
|---------|---------|
| **视觉档案** | 每个角色建独立的参考图文件夹 |
| **配色区分** | 3 人以上团队用不同主色调区分角色 |
| **身高差** | 在提示词中明确身高关系（"角色A比角色B高半个头"） |
| **站位固定** | 同一个场景中，角色站位关系不变（"A永远在左，B永远在右"） |
| **对白记录** | 建一张表，记录每个角色的对白风格（用词习惯、语气词、口头禅） |
