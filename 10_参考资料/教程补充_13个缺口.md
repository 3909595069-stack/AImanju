## 缺口1：真人照片→AI风格化转绘完整管线

林夕角色的特殊之处：**真实照片→AI风格化→保持"像本人"但"像动漫角色"**

### 操作步骤

**Step 1：拍摄林夕的原始素材**
- 5个角度：正面 / 3/4右侧 / 3/4左侧 / 侧面 / 全身站立
- 统一条件：自然光、纯白背景（或宿舍白墙）、同一套衣服（牛仔裤+白T恤作为定妆装）
- 表情：中性表情为主+微笑一张+认真表情一张
- 手机拍摄即可，但注意光线均匀，不要阴阳脸

**Step 2：上传即梦，风格化转绘**

即梦(jimeng.jianying.com)→图片生成→上传参考图→输入提示词：

```
年轻男性，20岁，[描述林夕的真实长相特征]，
穿着洗旧的白色T恤、深蓝色牛仔裤、白色帆布鞋，
二次元赛璐璐风格，新海诚风格光影，
半写实动画质感，角色设定图，全身站立，纯白背景，9:16
```

**关键技巧**：
- "半写实"——保证还像本人（纯二次元会丢掉面部特征）
- 上传的参考图与提示词的权重平衡：即梦里设"参考强度"为70-80%
- 多生成5-8次，选最像本人且最好看的那张
- 记录这张图的seed值（即梦生成后点击图片可查看）

**Step 3：扩展为角色身份证**

用同一seed+同人照片参考，换提示词中的角度关键词：
- "正面站立" → "侧面站立（身体朝右）" 
- → "背对镜头，背面站立" 
- → "面部特写，从胸口到头顶"

**Step 4：检验标准**
- 找5个不认识林夕的人看定妆照，再给他们看林夕真人照片——"能认出来吗？"
- 如果5人中少于3人能认出，降低风格化程度（"半写实动画质感"→"写实风格，轻微漫画滤镜"）

## 缺口2：静态漫剧+微动效MVP技术方案

你们第一版不作完整AI视频，而是"动态漫画"。

### 技术栈（全部国内可用、全部免费）

| 环节 | 工具 | 做什么 |
|------|------|--------|
| 静态关键帧 | 即梦4.0 | 生成每个分镜的高质量图片 |
| Ken Burns效果 | 剪映专业版 | 对图片做缓慢推拉横摇（模拟运镜） |
| 脸部微动 | LivePortrait | 让角色眼睛/嘴巴/头部微动 |
| 环境粒子 | 剪映特效 | 叠加光点/落叶/雾气 |
| 转场 | 剪映 | 0.5秒黑屏+whoosh音效 |

### Ken Burns参数（剪映关键帧）

| 镜头类型 | 缩放范围 | 位移方向 | 时长 |
|----------|----------|----------|------|
| 特写/近景 | 102%→105% | 轻微上下 | 3-4秒 |
| 中景 | 100%→104% | 轻微水平 | 4-5秒 |
| 全景/远景 | 100%→108% | 水平+垂直 | 5-6秒 |
| **黄金法则** | **缩放不超过8%** | **位移不超过画面10%** | **4秒最优** |

超过这些参数=观众感知到"生硬的缩放"，出戏。

### LivePortrait部署（开发同学负责）

1. GitHub: github.com/KwaiVGI/LivePortrait
2. 最低要求：GTX 1060 6GB显存
3. 一张源图（角色定妆照）→ 一段驱动视频（随便录一段自己的脸说话/眨眼/转头）→ LivePortrait把角色的脸驱动起来
4. 输出带透明通道的视频 → 导入剪映叠到静态图上
5. 这是从"静态漫剧"升级到"会呼吸的漫剧"最关键的一步

### 环境粒子（剪映操作）

剪映→特效→搜索：
- "光点飘落" → 架空世界氛围
- "萤火虫" → 夜晚场景
- "金粉" → 力量激活场景
- "雾气" → 悬疑/过渡场景
- "树叶飘落" → 校园日常

粒子叠加后调低不透明度到30-50%，太浓会像PPT特效。

## 缺口4：竖屏9:16构图专项指南

### 竖屏漫剧的构图铁律

| 构图类型 | 规则 | 为什么 |
|----------|------|--------|
| 单人特写 | 人脸占画面1/3宽度，居中偏上 | 竖屏人脸是绝对焦点 |
| 单人全身 | 人物占画面3/4高度 | 留顶上空间放字幕 |
| 双人对话 | **上下排列或前后景**，不能左右并排 | 竖屏宽度不够，左右并排人物太小 |
| 全景场景 | 利用竖屏的"高"展示建筑垂直感 | 茶山/教学楼的竖向构图天然震撼 |
| 字幕位置 | 画面下方1/5区域，黑底半透明衬底 | 不遮挡角色脸部 |
| 视线方向 | 角色看向的方向留更多空间 | 不"撞墙" |

### 双人对话的竖屏方案

**方案A：正反打（推荐，适合3分钟以下）**
- 镜头1：A说台词→A的近景
- 镜头2：B反应→B的近景
- 镜头3：切回A
- 优点：不需要多人同框，一致性最好
- 缺点：镜头数翻倍

**方案B：前后景排列**
- A在前景（占画面1/2，虚化边缘），B在后景（占画面1/3）
- 适合A为主、B为辅的对话
- Prompt: "two people, person A in foreground (close, slightly blurred edges), person B in background, vertical composition 9:16"

**方案C：分层合成（最高质量）**
- 单独生成A的图层+单独生成B的图层+单独生成背景
- 剪映里合成
- 工作量×3，但一致性100%

## 缺口5：多角色同框技术方案

### 你们项目的角色同框场景预估

第1季6集中最可能的多角色场景：
- 三人对话(林夕+瑞樟+同文)：第2-3集约3-5个镜头
- 四人集结(全员)：第5集约1-3个镜头
- Boss战(林夕+瑞樟+同文+顺城 vs 磐岩)：第6集约3-5个镜头

### 分层合成法操作流程（推荐，零成本）

```
1. 生成背景层：场景母代码→即梦生成纯背景图
2. 生成角色A层：A的定妆照做参考+动作描述→即梦生成A的独立图(透明背景用remove.bg)
3. 生成角色B层：同上
4. 生成角色C层：同上
5. 剪映合成：背景垫底→A左→B右→C中后→调整大小和位置→添加统一光影覆盖层
6. 加环境特效粒子统一融合
```

### 即梦多参考图方案（快捷但一致性稍低）

即梦支持最多12个并发参考文件：
1. 上传所有出场角色的定妆照
2. 上传场景母版图
3. 提示词写清楚每个角色的空间位置
4. 多生成8-10次，选最好的

### 避免多人同框的技巧（偷懒法）

用剪辑规避：
- 一个全景镜头（所有人都在，分层合成做）→ 然后正反打切单人近景
- 观众只需要一个全景知道"大家都在场"，后面的对话用正反打完成
- 第6集Boss战同理：一个全景分层合成→后续动作场景用单人+磐岩的特写交替

## 缺口6：色彩前后一致性管理系统

### 全局LUT设置

在剪映专业版中：
1. 第1集成片后→调节→导出LUT
2. 后续所有集导入后→应用同一个LUT
3. LUT参数建议：轻微提暖(色温+3~+5)、饱和度+5~+8、对比度+3~+5

### 角色色彩锚定表

| 角色 | 主色 | 辅色 | 点缀色 | 禁止色 |
|------|------|------|--------|--------|
| 林夕 | 白+牛仔蓝 | 灰色 | 🚫无(纯人类) | 金/紫(守护者专属) |
| 瑞樟 | 米白 | 深灰 | 金色(仅激活态) | 红/橙 |

| 磐岩 | 深灰(灰岩色) | 暗红(刻字色) | 金红(战斗态) | 绿/蓝 |

| 场景类型 | 主色调 | 色温 | 饱和度 | 氛围词 |
|----------|--------|------|--------|--------|
| 白天校园 | 暖黄+自然色 | 5500K | 正常 | 青春、明朗 |

### 每集的色调弧线

- 第1集(高能开局)：日常校园暖黄→异常事件冷蓝→被救后暖黄回归
- 第2-3集(日常)：全程暖黄+自然色为主
- 第4集(转折)：暖黄→逐渐偏冷→结尾第一次出现暗红
- 第5集(高潮前夜)：冷蓝+暗金为主
- 第6集(BOSS战)：暗红+金红→结尾回到暖黄(但比开头暗一点，暗示封印更危险了)

## 缺口11：实拍素材库建设指南

### 半天拍摄任务清单

**时间**：找一个晴天，上午8点出门，12点前拍完  
**装备**：手机(开最高画质)+充电宝  
**拍摄人**：摄影同学  

**拍摄路线**(按校园地理顺序)：

```
8:00-8:30  校训石(大学生创新创业园旁)
           全景(含一鉴桥背景)+中景+刻字特写+石面纹理
           
8:30-9:00  一鉴桥+天心湖
           桥全景+湖面+荷叶+栏杆+桥面
           
9:00-9:30  瑞樟教学楼
           建筑全景+入口+牌匾+走廊+教室内部(空教室)
           
9:30-10:00 同文教学楼
           建筑全景+入口+牌匾+走廊
            
10:00-10:30 顺城学生广场+四期食堂
           广场全景+店铺招牌+食堂内部+蜜雪冰城+益禾堂
            
10:30-11:00 荷花池
           全景+荷花近景+水面+池边小路
            
11:00-11:30 茶山(如有时间)
           远景+山腰小路+茶叶近景
```

**拍摄小提示**：

- 可以不追求"艺术感"，追求"清晰+端正+信息完整"
- 架空世界场景不需要实拍——全靠AI生成

## 缺口13：MVP快速验证最小可行管线

### 第一集MVP(目标：2天内出片)

**Day 1：素材生产**
```
上午(你+开发同学)：
1. 用DeepSeek生成第1集剧本初稿(用教程的五步提示词)
2. 你亲自写钩子和修改对白
3. 拆成8-12个分镜

下午(你)：
1. 即梦生成林夕定妆照(用真人照片风格化)
2. 即梦生成瑞樟/同文/顺城的定妆照(用教程万能公式)
3. 即梦生成校训石/教学楼/天心湖场景母版图

晚上(开发同学)：
1. 部署LivePortrait(如时间不够，第一集先用纯Ken Burns)
2. 测试即梦API/批量下载流程
```

**Day 2：后期+发布**
```
上午(摄影同学)：
1. 剪映导入12张关键帧图
2. 每张图加Ken Burns关键帧(参数参考缺口2)
3. 硬切剪辑，加0.5秒黑屏过渡
4. 剪映内置TTS配音
5. 加BGM+音效

下午(你)：
1. Final check，看一遍成片
2. 写标题+文案+封面
3. 摄影同学设计封面(醒图)

晚上：
🚀 发布！
```

### MVP验收标准(自己打分)

| 维度 | 及格线 | 目标线 |
|------|--------|--------|
| 画面清晰度 | 720P以上 | 1080P |
| 角色辨识度 | 能分清5个角色 | 林夕"像本人" |
| 声音清晰度 | 配音能听清 | 有情绪有语气 |
| 故事可懂度 | 观众能看懂在讲什么 | 观众想看下一集 |
| 节奏感 | 不太拖沓 | 有情绪起伏 |
| 总时长 | 1-2分钟 | 1.5分钟稳定 |

### 不要追求完美

- 第一集的目标是"发出去"，不是"完美"
- 有人看 = 成功
- 有人评论 = 大成功
- 有人追更 = 系列可以继续做
