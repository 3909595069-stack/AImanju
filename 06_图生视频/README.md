# 图生视频：让画面动起来

> "图生视频"就是把定稿的静态画面"注入生命"——这是最接近魔术的一步，也是最容易翻车的一步。

## 🔍 这是啥？

**图生视频**（Image-to-Video），就是把 [[../05_批量生图/README|批量生图]] 环节产出的关键帧图像，用 AI 视频生成工具转化为 2-5 秒的动态视频片段。

这里有一个关键认知：**图生视频不是"让图动起来"，而是"让图完成一个确定的微动作"**。

好的图生视频片段：主角慢慢抬头，眼神从失焦变为聚焦（一个动作，3秒）
差的图生视频片段：画面里所有东西都在乱动，像一个梦（没有焦点，没有目的）

## 💡 为啥做？

图生视频之所以是当前最优路线，核心原因：

1. **首帧可控**：你上传的图就是视频的第一帧——第一帧是对的你才能确保整个镜头是对的
2. **动作精度远高于文生视频**：AI 从一个静止画面"推演"运动，比从纯文字"想象"运动要准确得多
3. **连贯性**：相邻镜头可以用尾帧作为下一个镜头的首帧，实现自然的视觉衔接
4. **极致性价比**：图生视频比文生视频的 GPU 消耗低 40-60%。一张图的 prompt 成本 ≈ 一段视频的 1/10

> 文生视频目前还是"用 prompt 赌博"阶段。图生视频至少让赌注小了一个数量级。

## 📐 运镜控制系统工程

### 六大运镜方法（2025实战验证）

#### 方法一：首尾帧控制运镜法 ⭐⭐⭐（适用：即梦/可灵/Seedance）

**原理**：上传两张图（起点→终点），AI自动补全镜头运动轨迹。这是目前视频结构最稳定、人物变形最小的方案。

**操作流程**：
1. 生成首帧图（如角色在画面右侧30%、面向左）
2. 生成尾帧图（同一角色、同一场景，但角色在画面左侧70%、镜头变为俯视）
3. 在图生视频界面分别上传首帧与尾帧
4. 填写运镜提示词（如"横移环绕+拉升"）
5. 设置时长5秒、运动幅度中等

**适用**：武侠漫剧"横移+环绕+拉升"三合一运镜、希区柯克变焦

#### 方法二：参考图系统锁运镜法 ⭐⭐（适用：Seedance 2.0/可灵）

**原理**：上传角色定妆照（正面+侧面+全身）+ 场景图作为"参考文件"，AI提取特征后保持一致性。Seedance 2.0 最多支持12个并发参考文件。

**示例**：上传咖啡厅场景1张 + 女主角3张定妆照 → 输入"镜头一：女主角端杯；镜头二：男主推门；镜头三：两人对视"，系统自动生成三个分镜且角色一致。

#### 方法三：运动笔刷"画地为牢"法 ⭐⭐⭐（适用：即梦/Runway）

**原理**：使用运动笔刷涂抹主体并设置移动方向（或设为0保持静止），再涂抹背景设置运动方向，实现"主体分离运镜"。

**操作**：
1. 生成一张多人/主体的静帧底图
2. 运动笔刷中，涂抹人物 → 设置位移=0（锁定）
3. 全局 Camera Motion 设"推镜头"或"横移"
4. 配合提示词：`Parallax Effect, 视差效果，背景快速掠过，人物在中心保持静止`

#### 方法四：运动分数（Motion Score）控制法 ⭐⭐（开源模型适用）

**参数区间**：
- Motion Score 2-5：轻微运动（发丝飘动、睫毛眨动、对话微表情）
- Motion Score 10-20：动感叙事（环绕运镜、动作场面、情绪爆发）
- Motion Score >20：极易导致人物变形、画面崩坏，慎用

#### 方法五：参数面板机械臂法 ⭐⭐（适用：可灵/即梦）

**原理**：放弃纯文字描述运镜，使用 Camera Control 参数面板，拆解为 X轴(水平)/Y轴(垂直)/Z轴(变焦)/旋转。

**推荐参数（多人场景）**：
- 对话框：水平(+2) + 变焦(+1)
- 平移展示：水平(+5)
- 推近特写：变焦(+3)
- 所有参数控制在 **±2~5** 区间内（避免"融化效果"）

#### 方法六：分层透视合成法 ⭐⭐⭐⭐⭐（适用：剪映+LivePortrait）

**原理**：后期合成——"前景人物"与"背景运镜"分开生成，最后合成，100%解决人物穿插与变形。

**流水线**：
1. 可灵生成空镜头背景运镜视频
2. 即梦生成人物静帧 → LivePortrait 生成绿幕人物循环视频
3. 剪映中背景作为主轨，画中画叠加人物，关键帧控位置

### 运镜难度分级

| 难度 | 运镜方式 | 描述示例 | 适用场景 |
|:--|:--|:--|:--|
| ⭐ 简单 | **固定 + 微动作** | 人物微动、背景完全不动 | 对话、情绪特写、日常 |
| ⭐⭐ 中等 | **缓慢推近** | slow push-in toward subject's face | 情绪升级、关键时刻 |
| ⭐⭐⭐ 较难 | **缓慢横摇** | slow pan across the scene, left to right | 场景建立、环境展示 |
| ⭐⭐⭐⭐ 困难 | **跟拍 + 环绕** | camera follows and circles subject | 角色移动、转场过渡 |
| ⭐⭐⭐⭐⭐ 极难 | **复合运镜** | 推+摇+环绕组合 | 不推荐单人AI生成 |

### 运镜提示词万能结构

```
[主体描述] + [核心动作] + [运镜方式] + [光影氛围] + [风格标签] + [节奏控制]

实战范例：
"一位身穿深色风衣的侦探站在雨夜街头（主体），
烟头在雨中闪烁（动作），
镜头从下方缓慢仰拍（运镜），
霓虹灯在积水地面反射出彩色光晕（光影），
黑色电影风格、冷峻色调（风格），
节奏由静到紧张逐渐升温（节奏）"
```

## 🎯 首尾帧控制深度指南

首尾帧是图生视频的"杀手级功能"——精确控制镜头从什么画面开始、到什么画面结束。

### 基础策略

| 技巧 | 说明 | 关键参数 |
|:--|:--|:--|
| **相邻镜头衔接** | 镜头A的尾帧 = 镜头B的首帧（同一张图） | 过渡强度 0.65-0.8 |
| **情绪渐变** | 首帧：面无表情 → 尾帧：嘴角上扬 | 运动幅度：小 |
| **场景转场** | 首帧：门关着 → 尾帧：门半开 | 时长：3-5秒 |
| **希区柯克变焦** | 背景快速压缩但人物大小不变 | 运动幅度：轻微/Motion 5-8 |

### 进阶技巧

**变化控制**：首尾帧差距太大，AI 会"跳帧"而非"过渡"。变化控制在 **30% 以内**最稳定。

**提示词配合**：
```
首帧 → 尾帧 过渡时加入：
"smooth transition, gradual change, slow ease-in ease-out,
中间帧填补自然（natural in-between frame interpolation）"
```

**抽帧衔接法**：提取前一镜头的尾帧 → 作为后一镜头的首帧 → 确保角色位置/服装/光影完全一致。

### 首尾帧一致性参数

| 参数 | 推荐值 | 说明 |
|:--|:--|:--|
| **过渡强度（创造力/相关性滑块）** | 0.65-0.8 | 过渡自然且不易产生幻觉 |
| **运动幅度** | 0.4-0.6 | 避免过大导致首帧抖动和尾帧突变 |
| **首尾帧间隔** | 3-5秒 | 最长不超过6秒 |
| **首尾帧差异度** | <30% | 变化过大 = 跳帧而非过渡 |

## ⚡ 运动幅度与参数优化

### 通用参数黄金区间

| 参数 | 推荐设置 | 说明 |
|:--|:--|:--|
| **视频时长** | 4-6秒（最佳），最长10秒 | 超过6秒动作僵硬风险倍增 |
| **分辨率** | 先出 720P 测试 → 确认后 1080P | 节省时间 |
| **帧率** | 24fps（AI生成标准） | 后期可补帧到30fps |
| **运动幅度** | 对话/日常 0.3-0.5，动作 0.5-0.8 | >0.8 极易变形 |
| **首尾帧过渡强度** | 0.65-0.8 | 自然过渡 |
| **CFG Scale** | 7-8（图生视频专用） | 过高僵硬，过低跳帧 |
| **采样步数** | 25-35 | 帧间差异不超过5步 |
| **运镜指令数** | 1-3个 | 超过3个AI无法准确执行 |

### 按场景类型调节参数

| 场景类型 | 时长 | 运动幅度 | 运镜方式 | 提示词要点 |
|:--|:--|:--|:--|:--|
| **对话特写** | 3-4秒 | 0.3-0.4 | 固定+微动 | 背景静止、只描述一个微表情 |
| **日常动作** | 4-5秒 | 0.4-0.5 | 固定+缓推 | 动作具体、有始有终 |
| **情绪高潮** | 5-6秒 | 0.4-0.6 | 缓推+微环绕 | 画面留白、给情绪空间 |
| **动作打斗** | 3-4秒 | 0.6-0.8 | 首尾帧控制 | 一个动作一个镜头 |
| **场景建立** | 4-5秒 | 0.5-0.6 | 缓慢横摇 | 环境优先、无人物复杂动作 |

## 🎭 动作描述提示词工程

### 黄金规则

> **一段视频只描述一个动作。** 试图让 AI 做一连串复杂动作 = 必翻车。

### 动作描述万能模板

```
[主体] + [具体动作] + [速度/力度] + [情绪氛围]

✅ 好例子：
"女孩缓缓抬起头，眼神从手中的书移到镜头，嘴角微微上扬"
"雨水沿着窗玻璃向下滑落，窗外城市灯光模糊"
"风吹动窗帘，阳光从缝隙中洒入，尘埃在光柱中飘浮"

❌ 坏例子：
"女孩动" —— 太模糊，AI 不知道要做什么
"一场激烈的打斗" —— 动作太复杂，图生视频做不了
"所有东西都动起来" —— 没有焦点，画面会一片混乱
```

### 按动作类型提示词模板

| 动作类型 | 模板 | 示例 |
|:--|:--|:--|
| **微表情** | `[主体] 缓缓 [表情变化]，[细节描述]` | "女孩缓缓抬起眼睛，嘴角微微上扬，眼神从失焦变为明亮" |
| **日常动作** | `[主体] [动作]，[速度]，[环境互动]` | "男生慢慢端起咖啡杯，手微微抖动，杯沿碰到嘴唇" |
| **走跑移动** | `[主体] [方向] [移动]，[步态描述]` | "角色从左向右缓步行走，步伐轻盈，裙摆随步伐摆动" |
| **环境氛围** | `[自然现象] [运动方式]，[光线变化]` | "雨水沿窗玻璃向下滑落，窗外车灯扫过，光影在室内移动" |
| **转场过渡** | `[画面变化]，[过渡方式]，[衔接描述]` | "门缓缓关闭，画面渐暗，下一场景的光从门缝渗入" |

### 抽象词的"具象化翻译"

AI 在图生视频中不擅长理解抽象形容词。以下是常见抽象词的具象化替代：

| 抽象词 | 具象化替代 |
|:--|:--|
| "清冷高贵" | `挺直站姿，面无表情，目视前方，不与人目光接触` |
| "温婉动人" | `微微低头，嘴角带浅笑，眼睛弯成月牙形，双手交叠在身前` |
| "霸气外露" | `单手插兜，下巴微抬，眼神锋利，周围人退后半步` |
| "失落悲伤" | `肩膀下垂，头微微低下，眼神空洞，嘴角自然下垂` |
| "紧张不安" | `双手反复揉搓，四处张望，眉头紧锁，抿嘴唇，呼吸起伏加快` |

## 👥 多人场景与复杂镜头

### 多人同框不崩的原则

**在提示词中明确空间位置**：
```
"左边的角色A（黑发戴眼镜）正在低头看书，
右边的角色B（长发穿白衬衫）侧头看着A，
两人之间保持一个咖啡杯的距离，位置固定，高稳定性。"
```

**关键技巧**：
- 使用"保持角色间距不变"、"位置固定"、"高稳定性"等指令
- 大幅运镜必须通过**首尾帧控制法**或**分层合成法**实现
- 严禁一键生成大幅度甩镜头——几乎必崩
- 多人场景运动参数（水平/垂直/变焦）控制在 **±2~5**

### 多人场景解决方案对比

| 方案 | 适用 | 效果 | 难度 |
|:--|:--|:--|:--|
| **即梦参考图系统** | 2-3人对话 | 70-80% 一致 | ⭐⭐ |
| **Seedance 2.0 多参考文件** | 3-5人场景 | 80-90% 一致 | ⭐⭐ |
| **可灵首尾帧控制** | 任何人数 | 85-95% 一致 | ⭐⭐⭐ |
| **分层合成法** | 任何人数 | 95-100% 一致 | ⭐⭐⭐⭐⭐ |
| **Vidu Q1 多主体** | 最多7个主体 | 85-90% 一致 | ⭐⭐ |

### 2025 前沿：阿里 Tora2 多主体控制

阿里云 Tora2（ACM Multimedia 2025 Oral）实现了业界首次**多主体外观与运动端到端联合控制**。核心技术：
- **解耦个性化提取器**：ArcFace 捕捉人脸五官，独立纹理通道处理物体细节
- **门控自注意力绑定机制**：为每个主体建立专属"指令通道"，避免动作错配
- **运动-外观对比学习**：消除身份漂移与动作失真

## 🚫 图生视频翻车大全与排错

### 八大经典翻车问题

| # | 问题 | 典型表现 | 根因 | 标准解法 |
|:--|:--|:--|:--|:--|
| 1 | 🌊 **画面闪烁** | 画面区域不稳定地闪烁/抖动 | 逐帧独立生成+噪声不一致 | 降CFG到7-8/启用光流/统一噪声初始化/多重生成选最优 |
| 2 | 🫠 **人物变形** | 面部扭曲、肢体融化、瞬移 | 运动幅度过大 | 降运动幅度到0.4-0.6/缩短时长到3-5秒/首尾帧控制 |
| 3 | 🔄 **身份漂移** | 同一角色不同镜头像换了人 | 未锁定参考图 | 上传角色定妆图作为参考/使用Character Elements/LoRA触发词 |
| 4 | 🌀 **背景扭曲** | 背景变成"一锅粥" | 复杂运镜+背景纹理细密 | 避开细密纹理/固定机位优先/运动笔刷锁背景 |
| 5 | ⚡ **动作瞬移** | 人物从一个位置跳到另一个 | 帧间运动估算失败 | 增加关键帧密度/使用首尾帧控制/降低运动幅度 |
| 6 | 🎭 **表情突变** | 表情在视频中途突然变化 | AI"创造性"补充表情变化 | 提示词明确"表情保持不变"/参考图使用中性表情 |
| 7 | 🐛 **字幕/UI乱码** | 图中有文字时文字扭曲乱跳 | AI不理解文字的空间稳定性 | 生图时避免图中出现文字/后期在剪辑软件添加 |
| 8 | 🗑️ **AI抽搐帧** | 视频开头/结尾0.5秒画面异常 | 模型预热/冷却阶段不稳定 | **每个片段删除开头0.5s和结尾0.5s**——这是铁律 |

### 画面闪烁终极排错流程

```
Step 1: 缩短时长（成功率80%）→ 降到3-5秒
Step 2: 降低运动幅度（成功率70%）→ motion 0.3-0.5
Step 3: 降低CFG Scale（成功率65%）→ 7-8
Step 4: 避免细密纹理（成功率60%）→ 格子/条纹/百叶窗
Step 5: 切换模型版本（成功率50%）→ Seedance 2.0 vs 3.0
Step 6: 多重生成选最优（成功率80%/8版）→ 闪烁差异很大
Step 7: 抽取稳定帧+后期补帧（成功率70%）→ RIFE/FILM
```

### 多人物场景崩溃诊断

```
检查清单：
□ 是否在使用参考图/首尾帧控制？
□ 运动参数是否控制在 ±2~5？
□ 每个角色位置是否在提示词中明确描述？
□ 服装/发型/瞳色关键词在每个镜头是否一致？
□ 是否使用了"位置固定""高稳定性"等稳定指令？
□ 复杂镜头是否用了分层合成法？
```

## 📐 镜头衔接与转场设计

### 四大"相似性"转场法（让观众察觉不到剪辑点）

| 类型 | 手法 | 示例 |
|:--|:--|:--|
| **形状相似** | 相同几何形状的物体衔接 | 圆月→圆形钟表盘→圆形瞳孔 |
| **运动相似** | 运动方向和速度延续 | 赛车向右冲出→猎豹向右疾驰 |
| **位置相似** | 主体固定在画面同一位置 | 瞳孔位于画面中央→星球漩涡在画面中央 |
| **概念相似** | 同一主题词串联不同画面 | "破碎"→玻璃碎、照片撕碎、瞳孔裂纹 |

### 镜头组接七大原则

| 原则 | 说明 | 违规后果 |
|:--|:--|:--|
| **1. 动接动/静接静** | 运动镜头接运动镜头，静止接静止 | 视觉跳跃、观众眩晕 |
| **2. 30°原则** | 同一主体两镜头机位变化≥30° | 画面不流畅"跳一下" |
| **3. 180°轴线规则** | 对话场景摄影机不跨越虚拟轴线 | 人物朝向混乱 |
| **4. 动作匹配** | 前一镜的动作方向在后一镜延续 | 动作断裂 |
| **5. 视线匹配** | A看向画外的方向 = B被看的方向 | 视线不交汇出戏 |
| **6. 景别渐变** | 相邻镜头景别差1-2级 | "跳切"突兀感 |
| **7. 信息递进** | 每切一个镜头必须给观众新信息 | 重复信息=废镜头 |

### 图生视频专用转场技巧

| 技巧 | 实现方式 | 适合 |
|:--|:--|:--|
| **首尾帧无缝衔接** | 镜A尾帧=镜B首帧 | 日常叙事 |
| **遮挡转场** | 用物体遮挡镜头形成自然黑屏 | 场景切换 |
| **J-cut 听觉引导** | 画面切换前0.5秒先出现下一场景声音 | 所有类型 |
| **黑屏+音效** | 0.3-0.5秒黑屏 + 音效过渡 | 场景大切换 |
| **微停顿定格** | 关键高潮点设置0.3-0.5秒定格 | 情绪高潮 |

> **新手黄金法则**：你的前10部作品，所有场景切换用**黑屏 0.5秒 + 音效**。这不是保守——这是让观众喘气的必要空间。

### 镜头时长与叙事节奏

| 节奏类型 | 每镜头时长 | 适用场景 |
|:--|:--|:--|
| **快节奏（紧张/兴奋）** | 1-2秒 | 追逐、打斗、悬疑 |
| **中节奏（叙事推进）** | 2-4秒 | 日常对话、剧情推进 |
| **慢节奏（情绪沉淀）** | 5-8秒 | 抒情、告别、深刻反思 |
| **情绪重音** | 比其他镜头多1-2秒 | 关键时刻的情绪停留 |
