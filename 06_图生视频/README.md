# 图生视频：让画面动起来

> "图生视频"就是把定稿的静态画面"注入生命"——这是最接近魔术的一步，也是最容易翻车的一步。

## 🔍 这是啥？

**图生视频**（Image-to-Video），就是把 [[../05_批量生图/README|批量生图]] 环节产出的关键帧图像，用 AI 视频生成工具转化为 2-5 秒的动态视频片段。

这里有一个关键认知：**图生视频不是"让图动起来"，而是"让图完成一个确定的微动作"**。

好的图生视频片段：主角慢慢抬头，眼神从失焦变为聚焦（一个动作，3秒）
差的图生视频片段：画面里所有东西都在乱动，像一个梦（没有焦点，没有目的）

## 💡 为啥做？

图生视频之所以是当前最优路线，核心原因：

1. **首帧可控**：你上传的图就是视频的第一帧——第一帧是对的你才能确保整个镜头是对的
2. **动作精度远高于文生视频**：AI 从一个静止画面"推演"运动，比从纯文字"想象"运动要准确得多
3. **连贯性**：相邻镜头可以用尾帧作为下一个镜头的首帧，实现自然的视觉衔接

> 文生视频目前还是"用 prompt 赌博"阶段。图生视频至少让赌注小了一个数量级。

## 🛠️ 即梦图生视频操作步骤

1. **进入图生视频**：即梦 App → 底部 "+" → 选择"图生视频"
2. **上传首帧**：选择 [[../05_批量生图/README|批量生图]] 中定稿的关键帧图片
3. **（推荐）上传尾帧**：如果你想要精确控制镜头结束状态，上传尾帧图——这是控制镜头衔接的关键
4. **填写动作描述**：用下面的"动作描述模板"写一段简短的动作说明
5. **设置时长**：新手建议 3 秒，等你对工具熟悉后尝试 5 秒
6. **运镜设置**：即梦支持固定/推/拉/摇等基础运镜选项
7. **生成 + 筛选**：通常生成 2-4 个版本，选最好的一个
8. **（关键）检查画面闪烁**：看视频有没有画面闪烁/变形/人物突变——有就重新生成

### 动作描述模板

```
[主体] [具体动作] + [速度/力度] + [情绪氛围]

# 好例子：
"女孩缓缓抬起头，眼神从手中的书移到镜头，嘴角微微上扬"
"雨水沿着窗玻璃向下滑落，窗外城市灯光模糊"
"风吹动窗帘，阳光从缝隙中洒入，尘埃在光柱中飘浮"

# 坏例子：
"女孩动" —— 太模糊，AI 不知道要做什么
"一场激烈的打斗" —— 动作太复杂，图生视频做不了
"所有东西都动起来" —— 没有焦点，画面会一片混乱
```

> **黄金规则**：一段视频只描述**一个动作**。试图让 AI 做一连串复杂动作 = 必翻车。

## 📊 运镜原则

### 推荐运镜（从最简单到最难）

| 难度 | 运镜方式 | 描述示例 | 适用场景 |
|:--|:--|:--|:--|
| ⭐ 简单 | **固定 + 微动作** | 人物微微动，背景完全不动 | 对话、情绪特写、日常动作 |
| ⭐⭐ 中等 | **缓慢推近** | slow push-in toward subject's face | 情绪升级、揭示、关键时刻 |
| ⭐⭐⭐ 较难 | **缓慢横摇** | slow pan across the scene, left to right | 场景建立、展示环境 |
| ⭐⭐⭐⭐ 困难 | **跟拍** | camera follows subject walking | 角色移动、转场过渡 |
| ⭐⭐⭐⭐⭐ 极难 | **复杂运镜** | 推+摇组合/环绕 | 不推荐在 AI 短片中尝试 |

### 避免的运镜陷阱

| 避免 | 原因 |
|:--|:--|
| 快速推拉 (zoom) | 画面扭曲、变形、模糊 |
| 剧烈抖动 (shake) | AI 生成的"抖动"和真实的 camera shake 是两回事——它会变成画面撕裂 |
| 快速旋转 (spin) | 背景会变成一锅粥 |
| 同时做多个动作 | AI 会把所有动作都做一半，没有一个做完的 |

> **新手黄金法则**：你的前 10 部作品，所有镜头用固定机位。这不是保守——这是尊重观众的眼睛。

## 🔧 首尾帧控制技巧

首尾帧是图生视频的"杀手级功能"——它让你可以精确控制一个镜头从什么画面开始、到什么画面结束。

| 技巧 | 说明 |
|:--|:--|
| **相邻镜头衔接** | 镜头 A 的尾帧 = 镜头 B 的首帧（同一张图），确保无缝切换 |
| **情绪渐变** | 首帧：面无表情 → 尾帧：嘴角上扬，AI 会自然补间 |
| **场景转场** | 首帧：门关着 → 尾帧：门半开，用于转场过渡 |
| **不要跨度过大** | 首尾帧差距太大，AI 会"跳帧"而非"过渡"。变化控制在 30% 以内 |

## 🐛 画面闪烁解决方案

画面闪烁（flickering）是图生视频最常见的质量问题——画面某些区域在不稳定地闪烁/抖动。

| 原因 | 解决方案 |
|:--|:--|
| 首帧图本身有噪点 | 在生图时避免加 `film grain` 类关键词；如果原图有噪点，先在修图软件中降噪 |
| 动作描述太模糊 | 把动作写得更具体，给 AI 更多"锚定"信息 |
| 复杂纹理闪烁（如格子衬衫、细条纹） | 生图时避免过于细密的重复纹理；如有必要，用 PS 稍微模糊化纹理区域 |
| 即梦模型版本问题 | 尝试切换到即梦的不同模型版本（如 3.0 vs 4.0） |
| 背景复杂 + 人物动作 | 使用"背景静止 + 仅人物微动"的描述策略 |
| 根本解决：增加生成次数 | 同样参数生成 4-6 个版本，闪烁问题在版本间差异很大 |
