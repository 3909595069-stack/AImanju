# 🔒 场景一致性深度锁定技术

> 光有母代码还不够。当你切换镜头角度、焦距、景别时，场景仍然可能"漂移"。以下是对付漂移的进阶手段。

## 一、种子（Seed）控制的正确姿势

### 常见误区

❌ 锁定 Seed = 锁定一切，改提示词也没关系。
✅ **锁定 Seed ≠ 不变。** 随着提示词为了剧情而微调，固定种子也**会产生特征偏移**。Seed 只是随机起点，不是铁锁。

### 正确流程

1. 生成满意的第一张场景图 → **记录 Seed 值**（在即梦里点开图片详情可以找到）
2. 生成同一场景其他镜头时 → 使用相同 Seed → **只改角色描述和镜头角度**
3. 如果改动超过 30%（比如从室内切到窗外）→ 放弃同一个 Seed，重新生成后再记录新 Seed

### 不支持 Seed 的工具怎么办？

每次**复制粘贴相同的场景母代码**，确保除视角外的所有参数完全一致：

| 要改的 | 不改的 |
|--------|--------|
| 从"正面视角"→ 只改成"左侧 45 度视角" | 光源方向、色调、标志物、空间描述 |
| 从"中景"→ 只改成"远景" | 同上 |
| 从"女主走"→ 只改成"女主站" | 同上 |

## 二、ControlNet 深度图锁定（ComfyUI 用户专属）

如果你用 ComfyUI / Stable Diffusion：

1. **生成场景母版图**（高分辨率）
2. **提取深度图**：ControlNet → Depth Map（Midas / Zoe 模型）
3. **后续同场景生成**：加载这个深度图到 ControlNet → 权重 0.7-0.9
4. **结果**：AI 生成的每一帧共享**同一个物理空间结构**

> 这是目前个人创作者能用的**最强的场景一致性技术**——地板纹理、墙壁裂缝、窗户形状在每一帧中都精确一致。

## 三、多视角底图库

对每个核心场景，预生成以下视角的底图作为参考：

| 类型 | 镜头描述 | 什么时候用 |
|------|---------|-----------|
| **全景（Wide / Long）** | `wide establishing shot, the entire room is visible` | 新场景的第一个镜头 |
| **中景（Medium）** | `medium shot, waist up framing` | 对话、日常互动 |
| **过肩（OTS）** | `over the shoulder shot` | 两人对话 |
| **低角度** | `low angle looking up` | 角色显得强大/有威胁 |
| **高角度** | `high angle looking down` | 角色显得弱小/被压迫 |
| **细节（Detail）** | `extreme close up, surface detail` | 锁定材质纹理参考 |

## 四、工业化"场景库"建设

### 为什么需要场景库？

你的剧本 **不是只有一部剧**。当你开始做第二部、第三部时，你会发现很多场景可以复用。一个"温馨卧室"可以出现在甜宠剧、悬疑剧、末日剧里——只要改色调和灯光方向。

### 怎么建场景库？

1. 每做一个新场景 → 保存 3 张图（全景 + 中景 + 细节）+ 母代码
2. 命名规则：`场景类型_风格_编号`
   - `Cafe_Vintage_01`（复古咖啡馆1号）
   - `Bedroom_Cozy_02`（温馨卧室2号）
3. 存入 Obsidian 的 `_assets` 目录，在 `README` 里建索引
