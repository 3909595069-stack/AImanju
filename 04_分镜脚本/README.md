# 分镜脚本：从写手变导演

> 会写故事只是及格线，会"拆镜头"才是导演——这一步把文字翻译成画面语言。

## 🔍 这是啥？

**分镜脚本**，就是把一段文字剧情拆解为一个个具体的镜头指令。每个镜头明确告诉 AI：**看什么、怎么看、动不动的、看多久**。

它是一座桥：一边是你的文本剧本，另一边是 AI 能执行的画面指令。

对比一下：
- 剧本写的是："主角走在雨中，心情低落"
- 分镜写的是："中景，固定机位，主角从画面左侧入画向右侧行走，背景是模糊的城市夜景，冷蓝灰色调，雨丝在路灯下反光，面部微表情：嘴角下撇，眼神失焦，持续 3 秒"

## 💡 为啥做？

因为 AI 不懂"戏剧"，它只懂"指令"。

你给 AI 写"一个悲伤的镜头"，它会随机出一种悲伤——但你想要的悲伤是"中景面部特写，眼角有泪，嘴角在克制地颤抖"，还是"远景背影，人物在空旷的长街尽头慢慢变小"？

**不写分镜 = 把叙事权拱手交给掷骰子。**

而且，分镜还解决了另一个核心问题：**节奏**。什么时候给特写制造紧张，什么时候给全景让人喘口气——节奏是短片的灵魂，而节奏就写在分镜里。

## 🛠️ 怎么做？

### 8 要素法：每个镜头必须明确的 8 个维度

| # | 要素 | 说明 | 示例 |
|:--|:--|:--|:--|
| 1 | **景别** | 看多近 | 特写 / 近景 / 中景 / 全景 / 远景 / 大远景 |
| 2 | **角度** | 从哪看 | 平视 / 俯视 / 仰视 / 过肩 / POV |
| 3 | **运镜** | 动不动 | 固定 / 推 / 拉 / 摇 / 跟 / 旋转 |
| 4 | **构图** | 怎么放 | 三分法 / 中心 / 对角线 / 框架构图 / 留白 |
| 5 | **人物** | 谁在画面里 | 单人 / 双人 / 群体；角色面向何方；在画面什么位置 |
| 6 | **动作/表情** | 在做什么 | 具体动作描述 + 面部情绪关键词 |
| 7 | **光影/色调** | 什么氛围 | [[../03_场景设计/README|场景母代码]]中的光源和色调 |
| 8 | **时长** | 停多久 | 以秒为单位的关键帧描述（影响视频生成时的持续时间） |

### 景别表：6 级景别

| 景别 | 框取范围 | 叙事功能 | 适合的情绪 |
|:--|:--|:--|:--|
| **大远景** (ELS) | 人物极小，环境为主 | 建立世界观，史诗感，孤独感 | 渺小、自由、史诗、离别 |
| **远景** (LS) | 人物全身 + 大量环境 | 交代空间关系，过渡 | 进入新场景、旅程开始 |
| **全景** (FS) | 人物全身，少量环境 | 展示人物全貌和姿态 | 角色亮相、动作展示 |
| **中景** (MS) | 膝盖/腰部以上 | 最常用的叙事景别 | 对话、日常行动 |
| **近景** (CU) | 胸部以上 | 强调表情和情绪 | 内心变化、关键台词 |
| **特写** (ECU) | 面部局部 / 手 / 物件 | 强化细节，制造紧张 | 情绪高潮、关键线索、转折 |

### 黄金配比：30 / 50 / 20

一部 3 分钟的短片，建议的景别配比：

| 景别 | 占比 | 时长（3分钟片） | 作用 |
|:--|:--|:--|:--|
| 特写 + 近景 | **30%** | ≈ 54秒 | 情绪锚点——让观众"进入"人物 |
| 中景 | **50%** | ≈ 90秒 | 叙事主力——讲清楚在发生什么 |
| 全景 + 远景 | **20%** | ≈ 36秒 | 呼吸空间——开场建立 + 过渡/结尾 |

> **新手最容易犯的错**：全片中景从头到尾。因为中景最"安全"。但全片中景 = 没有任何情绪起伏，观众 30 秒就划走了。

### 运镜表

| 运镜 | AI 提示词关键词 | 情绪效果 | 建议 | 避免 |
|:--|:--|:--|:--|:--|
| **固定** (Static) | static shot, fixed camera | 冷静、观察、客观 | 对话、情绪酝酿 | 动作高潮 |
| **推** (Push-in) | slow push-in, dolly in | 注意力集中、紧张升级 | 关键台词、发现线索 | 频繁使用 |
| **拉** (Pull-out) | slow pull-out, dolly out | 揭示全貌、孤独感、落幕感 | 结尾、情绪释放 | 开场（除非刻意） |
| **摇** (Pan) | slow pan left/right | 展示空间、跟随运动 | 场景建立、展示环境 | 快节奏对话 |
| **跟** (Tracking) | tracking shot, follow | 陪伴感、代入感 | 角色行走、追逐 | 静态场景 |
| **固** (Fixed) | static camera, no movement | 稳定、不打扰叙事 | 绝大部分叙事镜头 | — |

> **新手黄金法则**：90% 的镜头用固定机位。你不是在拍动作大片，运镜越多越容易出错。保持克制。

## 🎯 一句话拆 8 个镜头的技巧

把剧本中的一句话分解为连续镜头，用这个框架：

```
原文："主角推开门，看到屋内一片狼藉，震惊地后退一步"

拆解：
镜1: [全景·固定·平视] 走廊中的主角，手搭在门把手上，深吸一口气
镜2: [特写·固定·俯视] 手拧动门把手，门缓缓打开一条缝
镜3: [中景·固定·平视] 门打开，主角的脸从门后露出，眼睛扫视屋内
镜4: [POV·摇镜] 从主角视角扫视屋内——翻倒的家具，碎玻璃，散落的照片
镜5: [特写·固定] 地上一个摔碎的相框，玻璃裂痕中的合影
镜6: [近景·固定] 主角的表情：眼睛睁大，嘴唇微张，呼吸急促
镜7: [中景·缓推] 主角踉跄后退一步，手扶住门框
镜8: [全景·固定] 主角背靠门框缓缓滑坐在地，身后是凌乱的房间
```

原始一句话 → 8 个镜头，节奏是：缓→紧→更紧→释放→重击→反应→后撤→定格。

## 🐛 新手常见错误

| 错误 | 为什么错 | 正确做法 |
|:--|:--|:--|
| 全片中景到底 | 没有景别变化 = 没有情绪起伏 | 用 30/50/20 配比强制自己切换景别 |
| 运镜过多 | 每个镜头都在推拉摇跟，观众晕 | 90% 固定机位，运镜只在关键情绪点使用 |
| 时长平均分配 | 每个镜头时长一样 = 机械感 | 紧张镜头 1.5-2s，对话 2-3s，抒情 4-6s |
| 忽略转场逻辑 | 上一个镜头和下一个镜头空间关系对不上 | 画一张简单的场景平面图，标出机位 |
| 镜头描述太抽象 | "一个悲伤的镜头"——AI 不理解 | 用景别+角度+运镜+情绪关键词的固定语法 |
| 没有建立镜头 | 开场直接怼脸，观众不知道在哪 | 第一个镜头 90% 应该是全景或远景 |

## 🤖 AI分镜提示词模板库

> 四个可直接使用的AI分镜生成提示词模板，覆盖基础拆解→专业六要素→中英双语→节奏规划。

### 模板一：结构分镜生成器（最简单）

```
你是一位专业的漫剧分镜编剧。请将以下故事生成为6格分镜剧本，每格需包含：
场景描述、角色动作、关键表情、镜头角度（平视/俯视/特写）。
故事梗概：[粘贴你的故事]
```

### 模板二：专业六要素分镜师（最全面）

```
【角色】你是一位资深影视分镜师，精通罗伯特·麦基的镜头拆解理论。
【任务】将小说剧本按独立动作单元拆解为分镜头方案。
【剧本原文】{粘贴你的剧本内容}

【分镜六要素】每个镜头聚焦单一动作：
1. 时间：[清晨/午后/深夜/具体时分+环境光效]
2. 地点：[场景名称+具体方位+空间特征]
3. 镜头设计：[景别]+[运镜：固定/推轨/摇移/手持]
4. 人物行为：单一动词指令，只描述[谁+做了什么/说了什么]
5. 画面结果：动作的即时后果，提供视觉证据
6. 观众情绪：[情绪类型]+[强度：↑↑↑/↑↑/↑/→/↓]+[落点]

【输出格式】
镜头[编号]·[景别] | [时间]·[地点] | 动作：[人物]+[行为] → 结果：[后果] | 情绪：[类型]([强度/落点])
```

### 模板三：中英双语分镜（面向MJ/SD/即梦）

```
你是一个专业的分镜师，根据用户提供的脚本文案，写一个纯文字分镜，一共[25]秒。
分镜不超过[12]个画面。
每个分镜要包含画面内容描述。
需要中英双版，中文和英文分开各自完整一段。

还需提供除了分镜提示词之外的通用提示词：
- 视频、美术风格：？

```

### 模板四：爽文节奏规划器

```

反派嘲讽/打压主角（冲突出现）→ 主角心理活动（金手指出现）

```

## 📊 工业化分镜Excel模板

> AI漫剧工业化批量生产的核心，是将分镜脚本**标准化为Excel表格**——作为AI生成工具的"施工图纸"。

### 标准分镜表（12列完整版）

| 字段 | 示例 | 用途 |
|------|------|------|
| **SHOT_ID** | `S03-07` | 唯一追踪编号 |
| **镜头时长** | `5s` | 节奏控制，可计算总时长 |
| **目的标签** | `冲突升级` | 强化叙事策略 |
| **画面一句话** | `逼近反派，雨打脸` | 叙事不跑偏的核心锚点 |
| **出镜角色** | `男主,反派` | 角色清单，防遗漏 |
| **角色状态** | `愤怒+逼近` | 表演可控，表情动作一致 |
| **表演强度** | `4/5` | 情绪梯度可视化 |
| **景别/角度** | `近景/仰拍` | 镜头语言明确 |
| **运动+强度** | `推进/轻` | 防止过度乱动 |
| **连续性约束** | `伤口在左脸` | 防止跨镜穿帮 |
| **参考图/资产ID** | `CHAR_001` | 防止角色变脸 |
| **输入类型** | `I2V` | 图生视频/文生视频 |

### 工业化五步流程

1. **定脚本**：用DeepSeek/豆包批量生成剧本
2. **建分镜表**：用AI生成标准化12列分镜表
3. **建素材库**：每个角色生成四视图+每个场景生成标准环境图
4. **批量生成视频**：画面比例统一9:16竖屏，优先I2V+参考图约束
5. **剪辑配音**：剪映批量拼接，AI配音

| 指标 | 传统方式 | AI工业化 |
|------|----------|----------|
| 团队规模 | 5-10人 | 1-3人 |
| 单集制作周期 | 1周 | 2-3小时 |
| 日产量 | 0.5集 | 5-10集 |
| 一个项目周期 | 数月 | 2-5天 |

## 🎵 分镜节奏与页面韵律

> 漫画分镜的节奏是**视觉音乐**。面板大小=时间感知，大格=慢（情绪沉淀），小格=快（紧张动作），无边框=时间冻结（闪回/白日梦）。

### 页面节奏模式

| 模式 | 面板组合 | 最佳用途 |
|------|---------|----------|
| **强-弱-强-弱** | 大-小-大-小 | 平稳对话、日常场景 |
| **弱-弱-弱-强!** | 小-小-小→满版大格 | 紧张积聚到爆发/高潮 |
| **递减模式** | 大→中→小→特写 | 逐渐聚焦、悬念逼近 |
| **递增模式** | 特写→中→大→满版 | 拉远揭示、规模拓展 |
| **均匀模式** | 全部等大 | 均匀时间流、日常流水 |

### Cohn视觉叙事语法 (Visual Narrative Grammar)

| 叙事角色 | 缩写 | 功能 | 类比 |
|---------|------|------|------|
| **建立者 Establisher** | E | 初始状态/环境建立 | 开头大远景 |
| **起始 Initial** | I | 打破平衡的触发事件 | 冲突开始 |
| **延续 Prolongation** | L | 紧张的持续/升级 | 对峙过程 |
| **高峰 Peak** | P | 情绪/动作的顶点 | 爆发瞬间 |
| **释放 Release** | R | 余波/结果/转折 | 结局回味 |

> 经典漫画页面 = E→I→L→P→R 循环，像一个完整的"叙事呼吸"。

## 👁️ 视线引导与阅读动线设计

> 漫画页面上角色的视线方向和对话框位置，共同决定了读者的**阅读路径**。

### 阅读动线基本原则

| 原则 | 说明 | AI实现方式 |
|------|------|-----------|
| **Z型路径** | 日漫右→左，美漫左→右，Z字扫描 | 面板排列方向在Prompt中明确 |
| **角色视线引导** | 角色眼神应看向下一个面板的方向 | Prompt加"looking right/left toward next panel" |
| **对话框即路标** | 对话气泡排列顺序=阅读顺序 | 生成画面不带文字，后期叠加 |
| **动作方向一致** | 右向运动=前进；左向运动=后退/回忆 | 关键场景统一方向 |
| **入口面板** | 页面左上角（日漫右上角）是视线落点 | 最重要信息放入口面板 |

### 对话框布局十种类型

| 类型 | 形状特征 | 功能 |
|------|---------|------|
| **普通对话** | 椭圆+尾巴 | 常规对话 |
| **思考气泡** | 云朵状 | 内心独白 |
| **旁白框** | 方形无尾 | 叙述性文字 |
| **耳语气泡** | 虚线边缘 | 悄悄话/秘密 |
| **喊叫气泡** | 锯齿/尖刺形 | 愤怒/高情绪 |
| **效果音气泡** | 异形+手写字 | 环境音效 |
| **电子音气泡** | 方形+数字字体 | AI/机器人 |
| **破碎气泡** | 裂纹/波浪边缘 | 虚弱/濒死 |
| **重叠气泡** | 双泡连接 | 快速对话/抢话 |
| **风格化气泡** | 定制形状 | 特殊角色语调 |

> 💡 AI漫剧最佳实践：生成画面时Prompt末尾加 `NO TEXT, no speech bubbles, no words`——画面保持干净，所有对话框和文字用剪辑软件后期叠加。

## 🎬 镜头组接七大原则

| 原则 | 说明 | 违规后果 |
|------|------|---------|
| **1. 动接动/静接静** | 运动镜头接运动镜头，静止接静止 | 视觉跳跃、观众眩晕 |
| **2. 30°原则** | 同一主体两个镜头机位变化至少30° | 画面"跳一下"不流畅 |
| **3. 180°轴线规则** | 对话场景摄影机不跨越人物虚拟轴线 | 人物朝向混乱、空间感崩塌 |
| **4. 动作匹配** | 前一镜的动作方向在后一镜中延续 | 动作断裂 |
| **5. 视线匹配** | A看向画外的方向=B在下一镜被看的方向 | 视线不交汇、出戏 |
| **6. 景别渐变** | 相邻镜头景别差1-2级 | "跳切"突兀感 |
| **7. 信息递进** | 每切一个镜头必须给观众新信息 | 重复信息=废镜头 |

> ⚠️ 180°轴线规则：两人对话时画一条虚拟线，所有镜头必须从同一边拍。AI解法：始终写明 `camera stays on the same side of the conversation axis`

## 📱 条漫(Webtoon)专用分镜技巧

> 条漫（竖屏滚动阅读）与传统翻页漫画有本质差异。手机屏幕约800px宽，读者在**上下滑动**中感受节奏。

### 条漫 vs 传统漫画

| 维度 | 传统翻页漫画 | 条漫 (Webtoon) |
|------|------------|---------------|
| **阅读方向** | 右→左（日漫）/ 左→右（美漫）| 上→下（垂直滚动）|
| **色彩** | 多为黑白 | 全彩，柔色调为主 |
| **节奏** | 面板密集，信息量大 | 更松散，更"电影化" |
| **面板边界** | 明确边框+间距 | 常无边栏，开放过渡 |
| **主要设备** | 印刷品 | 手机屏幕 |

### 六种条漫专属布局技巧

| 技巧 | 实现方式 | 效果 |
|------|---------|------|
| **扩展间距** | 面板间留2-3倍正常空白 | 时间流逝、悬念 |
| **全屏宽格** | 面板占满屏幕宽度(800-1000px) | 冲击力、场景切换 |
| **超长竖格** | 极端纵向拉长(2000px+) | 坠落感、压倒性尺度 |
| **快速剪辑** | 3-5个小格紧挨连发 | 快节奏动作 |
| **无声格** | 纯景色/表情，零文字 | 情绪呼吸空间 |
| **留白悬念** | 揭示前故意大量空白 | 积聚期待感 |

**条漫画幅规格**：宽度800-1080px，每集高度10000-20000px，每集30-60格，分辨率144-300 dpi。AI生成建议：9:16竖屏单格→后期拼接为长条。

## 🚫 分镜常见翻车与排错（扩展）

| 问题 | 症状 | 诊断 | 解决方案 |
|------|------|------|---------|
| **PPT感** | 每镜都是固定中景 | 景别和角度没变化 | 景别按2级差轮换 |
| **跳切突兀** | 远景直接切瞳孔特写 | 景别跨3级以上 | 中间加中景/近景过渡 |
| **动作断裂** | 前镜抬手后镜已放下 | 动作没匹配 | 分镜表加"连续性约束"列 |
| **越轴混乱** | 人物忽左忽右 | 违反180°轴线 | 固定摄影机在轴线同侧 |
| **信息重复** | 两个镜头讲同一件事 | 没有信息递进 | 每切一镜加新信息 |
| **情绪平坦** | 全片一个节奏 | 景别配比单一 | 高潮处特写+短镜，抒情处远景+长镜 |
| **动作堆砌** | 一个镜头塞3个动作 | 没遵守一对一 | 走路→停住→转身=3个镜头 |
